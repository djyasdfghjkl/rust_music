# 一、项目总览：Miku Tunes —— 初音未来主题桌面音乐播放器

## 1.1 项目定位

**项目名称**：Miku Tunes

**项目定位**：一款以初音未来（Hatsune Miku）为核心视觉IP的轻量级桌面音乐播放器，采用Tauri + Rust技术栈构建，实现纯本地音源解析与播放，不依赖任何后端服务，所有音乐数据保留在用户本地设备上。

**核心价值**：
- **隐私优先**：无需用户账号、不上传任何数据到云端，用户音乐始终保存在本地设备中
- **轻量高效**：基于Tauri框架打包的桌面应用，安装包极小，资源占用远低于Electron类应用
- **主题沉浸**：以初音未来视觉元素为核心打造沉浸式二次元风格音乐体验

## 1.3 核心功能模块详解

### 1.3.1 本地音源加载与扫描

Rust后端使用文件系统遍历 API，递归扫描用户指定的音乐文件夹，过滤常见音频格式文件。

**主要功能**：
- 用户可通过前端界面选择音乐文件夹
- 支持增量扫描，仅处理新增/修改的音乐文件
- 扫描进度实时反馈（当前文件数/总文件数）
- 支持排除指定文件夹（如缓存目录、下载临时目录等）

**文件格式支持**（由 Symphonia 提供支持）：
| 格式 | 解码支持 | 元数据 | 说明 |
|------|----------|--------|------|
| MP3 | Excellent | ID3v1/ID3v2 | 最广泛支持的音乐格式 |
| FLAC | Excellent | Vorbis comment | 无损音频，音质优秀 |
| WAV | Excellent | RIFF | 标准无损音频 |
| AAC | Great | ISO/MP4 | 高效率压缩格式 |
| OGG | Great | Vorbis comment | 开源压缩格式 |
| ALAC | Great | ISO/MP4 | Apple无损格式 |

> 说明：Symphonia 是一个纯 Rust 实现的音频解码框架，解码正确性与 FFmpeg 相当，性能差距通常在 ±15% 以内，完全足够桌面播放器使用。

### 1.3.2 元数据解析

基于 Symphonia 的 metadata 模块解析音频文件内的内置标签。

**采集字段**：
- 歌曲标题（Title）
- 艺术家（Artist）
- 专辑名称（Album）
- 专辑封面图片（Cover Art，提取并存储为本地缓存）
- 年份
- 音轨编号
- 时长（从音频流计算）
- 文件路径

### 1.3.3 本地数据库存储与查询

使用 Tauri SQL 插件 + SQLite 实现完全离线的音乐库管理。

**数据库表设计**：

```sql
-- 歌曲主表
CREATE TABLE songs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT NOT NULL,
    artist TEXT,
    album TEXT,
    year INTEGER,
    track_number INTEGER,
    duration INTEGER NOT NULL,
    file_path TEXT UNIQUE NOT NULL,
    cover_cache_path TEXT,
    play_count INTEGER DEFAULT 0,
    last_played INTEGER, -- timestamp
    added_time INTEGER NOT NULL  -- timestamp
);

-- 索引：加速常用查询
CREATE INDEX idx_title ON songs(title);
CREATE INDEX idx_artist ON songs(artist);
CREATE INDEX idx_album ON songs(album);
CREATE INDEX idx_file_path ON songs(file_path);

-- 播放列表表
CREATE TABLE playlists (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    create_time INTEGER NOT NULL
);

-- 播放列表关联表
CREATE TABLE playlist_songs (
    playlist_id INTEGER,
    song_id INTEGER,
    sort_order INTEGER,
    FOREIGN KEY (playlist_id) REFERENCES playlists(id),
    FOREIGN KEY (song_id) REFERENCES songs(id)
);

-- 收藏表
CREATE TABLE favorites (
    song_id INTEGER PRIMARY KEY,
    favorited_time INTEGER NOT NULL,
    FOREIGN KEY (song_id) REFERENCES songs(id)
);
```

**查询功能**：
- 通过 `title LIKE ? OR artist LIKE ?` 模式匹配实现模糊搜索
- 支持按歌曲标题、艺术家、专辑名称交叉查询
- 支持最近添加歌曲、最近播放歌曲、最多播放歌曲等多种筛选排序
- 支持数据库迁移，后续可平滑扩展功能

### 1.3.4 音频播放策略

Tauri 的 `asset://` 协议无法直接访问用户上传的文件，因此采用内置轻量级 HTTP 服务来处理音频流。

**播放实现方案**：
- Rust 后端启动一个仅监听 `127.0.0.1:随机端口` 的 HTTP 服务，提供音频文件访问端点
- 前端 HTML5 `<audio>` 元素直接请求后端 HTTP 流式端点进行播放
- 支持范围请求（Range Requests），实现播放进度拖拽功能
- 绑定操作系统媒体控制（Media Session API），支持键盘多媒体键控制

**播放器核心功能**：
- 播放/暂停/上一首/下一首
- 进度条拖拽跳转
- 音量调节与静音
- 播放模式切换：顺序播放、单曲循环、列表循环、随机播放

> 如果后续需要支持更复杂的音频处理需求（如均衡器），可以考虑在前端 Web Audio API 层面实现，音频流解码后通过 AudioContext 进行处理。

| 功能模块 | 计划内容 |

## 1.5 核心技术实现要点

### 1.5.1 元数据解析示例（Rust）

```rust
// 基于 Symphonia 的 ID3 标签解析示意
use symphonia::core::formats::FormatReader;
use symphonia::core::meta::MetadataOptions;
use symphonia::default::get_probe;

pub fn extract_metadata(file_path: &str) -> Result<SongMetadata, Box<dyn Error>> {
    let file = std::fs::File::open(file_path)?;
    let probed = get_probe().format(&file, &Default::default())?;
    let mut format = probed.format;
    let track = format.default_track()?;
    let tags = track.metadata().get();
    
    // 提取标题、艺术家、专辑等字段
    let title = tags.iter()
        .find(|t| t.tag == "title").and_then(|t| t.std_key_value().1);
    // ...
}
```

### 1.5.2 数据库查询示例（Rust + SQLite）

```rust
use tauri_plugin_sql::{Builder, Migration, MigrationKind};

// 初始化数据库与迁移
pub fn init_db(app: &mut tauri::App) -> Result<(), Box<dyn Error>> {
    let migrations = vec![
        Migration {
            version: 1,
            description: "创建歌曲表".to_string(),
            sql: include_str!("../migrations/001_create_songs.sql"),
            kind: MigrationKind::Up,
        },
    ];
    
    Builder::new("sqlite:miku_tunes.db")
        .add_migrations(migrations)
        .build()?;
}

// 模糊查询命令
#[tauri::command]
async fn search_songs(keyword: String, db: tauri::State<'_, sqlx::SqlitePool>) -> Result<Vec<Song>> {
    let pattern = format!("%{}%", keyword);
    let results = sqlx::query_as!(
        Song,
        "SELECT * FROM songs WHERE title LIKE ? OR artist LIKE ? OR album LIKE ?",
        pattern, pattern, pattern
    ).fetch_all(&*db).await?;
    Ok(results)
}
```

### 1.5.3 无边框透明窗口配置

在 `tauri.conf.json` 中进行如下配置，实现无边框透明窗口，为自定义初音未来主题设计提供基础：

```json
{
  "tauri": {
    "windows": [
      {
        "decorations": false,
        "transparent": true,
        "resizable": true,
        "width": 1200,
        "height": 800,
        "title": "Miku Tunes"
      }
    ]
  }
}
```

配置说明：`decorations: false` 禁用系统默认窗口装饰（标题栏和边框），`transparent: true` 允许窗口背景透明，配合自定义标题栏组件（`<div data-tauri-drag-region>` 实现拖拽区域）实现全定制化窗口外观。


# 二、主题风格设计 —— 初音未来二次元主题

## 2.1 设计理念

以初音未来角色为核心IP元素，融合 **苍绿 / 粉红 / 科技蓝** 的混合视觉风格，营造充满活力的二次元音乐世界氛围。

参考初音未来角色视觉特点：标志性的绿色双马尾、黑色与红色的方形发饰、蓝绿色饰边的白色服装、左肩的红色「01」标识。整体设计语言兼顾经典初音元素与现代化音乐播放器交互规范，打造从视觉到听觉的沉浸式体验。

  - 标题展示 "初音 Miku"
  - 正文展示 Miku 简介文字，采用淡入淡出渐变遮罩截断长文本
  - **"Read More"** 按钮：使用 Miku Pink 填充，悬停时颜色加深

- **"Artwork >" 横向滚动区**：
  - 横向滚动展示音乐专辑封面/角色插画缩略图
  - 采用 carousel 或横向滚动列表布局
  - 每个缩略图卡片配有简短的专辑名称

- **Recent Search 区域**：
  - 展示用户最近的搜索关键词
  - 包含关键词标记（chip）样式，点击后快速重新搜索
  - 参考图片中的样式，带有 "The Number / The Singer / The Song" 类似卡片推荐内容

  - 卡片式布局展示推荐内容（如 The Number / The Singer / The Song）
  - 每个卡片包含标题、简要描述和署名 "@作者名"
  - 卡片悬停时触发轻微上浮 + 光晕边框效果

### 2.3.4 底部播放器控制栏

- **左侧歌曲信息区**：专辑封面缩略图 + 歌曲名 + 艺术家名
- **中央控制区**：上一首 + 播放/暂停 + 下一首 + 播放模式切换
  - 播放/暂停按钮使用圆形渐变大按钮，主色调 Miku Pink
  - 控制按钮悬停时变为 Miku Green，并带有微妙的缩放动画
- **进度条区**：当前时间 + 进度条 + 总时长
  - 进度条轨道使用科技蓝渐变（Miku Blue 渐变），已播放部分使用 Miku Green，播放头为圆形小点
  - 支持拖拽进度条跳转
- **右侧控制区**：音量控制 + 播放列表切换
  - 音量滑条圆角设计，轨道和进度条配色与主进度条保持一致

### 2.3.5 字体系统

使用符合二次元风格的现代无衬线字体优先方案：

- **中文**：苹方（SF Pro） / 微软雅黑 / 思源黑体
- **英文/数字**：Inter / Poppins
- **强调数字**（如播放次数、歌曲时长）：JetBrains Mono
- **Logo/标题**：可考虑使用定制字体或 Miku 风格的手写字体


## 2.4 视觉特效与动效

| 特效类型 | 实现方式 | 触发场景 |
|----------|----------|----------|
| **悬浮上浮** | `transform: translateY(-4px)` + 阴影增强 | 卡片/按钮 hover |
| **脉冲波纹** | CSS keyframes 动态绘制放射状圆环 | 播放状态时的专辑封面 |
| **音乐频谱动画** | Web Audio API 获取实时频域数据，Canvas 动态绘制 | 正在播放歌曲时 |
| **玻璃模糊** | `backdrop-filter: blur(12px)` | 侧边栏、播放器栏、浮动面板 |
| **樱花飘落** | Canvas 粒子系统 | 可选，可独立开关提升性能 |
| **页面过渡** | Vue 路由切换淡入淡出 + 轻微位移 | 页面切换时 |
| **点击反馈** | 瞬间缩放 + 释放恢复 | 所有可交互元素 |

> 所有动效遵循「适度原则」，不干扰用户使用流畅度，动画时长控制在 150~300ms 之间。
