# Miku Tunes · 首页设计方案

## 一、设计目标

以初音未来视觉IP为核心，结合第二张参考图片的信息架构（歌曲推荐、热门艺人、即将演出等），打造一个**兼具二次元沉浸感与音乐播放器实用性的首页**。用户打开应用后能快速浏览推荐内容、访问常用模块，并感受到强烈的Miku角色氛围。

## 二、整体布局架构

采用三栏式布局，但左右不对称以突出视觉层次。参考图片左侧长列表+右侧双列信息，结合初音主题进行优化。

| 区域 | 宽度占比 | 内容模块 |
|------|----------|----------|
| **左侧边栏** | 20%（约240px） | 导航菜单 + 播放列表快速入口 + 当前用户信息 |
| **中央主内容区** | 55% | 横幅欢迎区 + Featured Songs 网格 + Popular Artist 横向滚动 |
| **右侧辅助区** | 25% | Recommended Songs 列表 + Upcoming Shows 卡片 + 近期播放记录 |

### 2.1 左侧边栏（沉浸导航）

- **顶部**：应用Logo（Miku Tunes），配以初音剪影图标。
- **主导航**：首页、曲库、搜索、收藏、设置 — 使用图标+文字，选中态为初音绿（#39C5BB）加左侧亮条。
- **播放列表快捷入口**：显示用户创建的播放列表（如“我最爱的Miku曲”、“Vocaloid精选”等），支持滚动。
- **底部**：当前播放队列的缩略信息，以及“本地音源扫描状态”。

### 2.2 中央主内容区（核心推荐流）

#### 2.2.1 顶部欢迎与角色展示区（Hero）

- **左侧文字**：根据时间段显示“下午好，Miku Fan” + 一句二次元风格的问候语（如“今天也想听Miku唱歌吗？”）。
- **右侧**：初音未来的小型角色立绘或圆形头像（使用您提供的图片），并显示“01”标识，点击可切换随机推荐歌单。

#### 2.2.2 Featured Songs（精选歌曲）

- 网格布局，每行3~4个卡片，支持横向滚动（如果数量多）。
- **每个卡片**：专辑封面占位（可用抽象波形或Miku相关装饰图），歌曲名（加粗）、艺术家名（次要文字）。
- **悬停效果**：卡片上浮，出现播放按钮（粉红色圆形），封面轻微发光。

#### 2.2.3 Popular Artist（热门艺人）

- 采用横向滑动列表（类似Carousel）。
- **每个艺人**：圆形头像（可使用渐变色+艺人首字母或抽象图标），下方显示艺人名。
- 点击艺人头像可快速筛选该艺人歌曲。
- 从参考图中提取艺人列表：Alan Walker, Taylor Swift, Dua Lipa, The Chainsmokers, Ariana Grande, Justin Bieber, Eminem, JoJo 等。

### 2.3 右侧辅助区（即时信息）

#### 2.3.1 Recommended Songs（推荐歌曲）

- 纵向列表样式，每行包含：歌曲名、艺术家、时长。
- 右侧增加一个“播放”图标或“+”图标（添加到播放列表）。
- 鼠标悬停时高亮整行，显示绿色播放按钮。
- 推荐内容可基于本地播放历史或随机展示（因无后端，可基于最近添加歌曲）。

#### 2.3.2 Upcoming Shows（即将到来的演出/活动）

- 卡片式列表，每张卡片显示活动名称、时间（如“Today’s Hits at 9:00pm”）、地点或主办方。
- 可增加“提醒我”按钮（本地通知占位）。
- 参考图片中“Today’s Hits”重复出现，可改为不同的Vocaloid线上演唱会或初音未来相关活动（如“Miku Expo 2025”、“Magical Mirai”等）增加主题贴合度。

#### 2.3.3 Recently Played（最近播放）

- 简单显示最近播放的3~5首歌曲，点击直接播放。
- 这是对右侧区域功能的补充，增强实用性。

## 三、视觉与动效设计要点（初音未来主题深化）

### 3.1 色彩应用

- **主背景**：深蓝紫渐变（#1A1A2E 到 #0F0F1E），营造夜场/科技感。
- **强调色**：初音绿（#39C5BB）用于选中态、进度条、按钮边框；粉红色（#FF9EC5）用于播放按钮、收藏图标、通知角标。
- **玻璃质感**：侧边栏和卡片采用半透明深色背景+背景模糊（backdrop-filter: blur(12px)），边缘勾勒细绿色描边。
- **文字层次**：纯白用于主标题，浅灰紫（#B8B8D6）用于辅助文字，降低视觉疲劳。

### 3.2 插图与装饰

- **背景点缀**：散布半透明的音符、双马尾轮廓、01数字矩阵，增加二次元氛围但不干扰阅读。
- **角色元素**：在Hero区域或右下角固定放置Miku的Q版剪影，随音乐播放产生轻微浮动。
- **专辑封面占位**：统一使用渐变几何图形+Miku的剪影或葱色元素，不依赖真实封面以保持风格统一。

### 3.3 动效规范

- **页面切换**：淡入+轻微向右位移（200ms），保持流畅。
- **卡片悬停**：上浮4px + 增加绿/粉外发光，过渡0.2s。
- **播放按钮点击**：脉冲缩放效果，配合Audio反馈。
- **进度条拖拽**：平滑跟随鼠标，实时更新时间戳。
- **音乐频谱动画**：仅在底部播放栏显示微型频谱条，使用初音蓝绿渐变色。

## 四、响应式与交互说明

- **窗口尺寸**：默认1200×800，支持最小宽度1000px，低于此宽度时右侧辅助区自动折叠为底部抽屉。
- **无边框窗口**：标题栏完全自定义，左上角显示Miku图标，右上角提供窗口控制按钮（最小化/最大化/关闭），中间区域可拖拽移动窗口。
- **键盘快捷键**：空格播放/暂停，左右箭头切换歌曲，上下箭头调整音量（设计文档中注明）。
- **本地查询**：首页搜索框位于标题栏，输入关键词后实时在前端对已加载的曲库数据进行过滤，无需后端请求。

## 五、首页模块的优先级排序

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0（必须） | Featured Songs 网格 | 用户第一眼看到的内容推荐 |
| P0 | 底部播放控制栏 | 全局播放控制 |
| P1 | Popular Artist 横向列表 | 快速筛选艺人 |
| P1 | Recommended Songs 列表 | 个性化推荐 |
| P2 | Upcoming Shows | 扩展内容，增加社区感 |
| P2 | 最近播放记录 | 提升便利性 |
| P3 | 欢迎+Hero角色区 | 品牌氛围，无功能性但强化IP |

## 六、与参考图片的差异改进

参考图片中的布局信息密度较高，但视觉层次不够鲜明。我们在设计中：

- 将“Popular Artist”从普通列表改为横向圆头像滚动条，更符合现代音乐应用习惯。
- 将“Recommended Songs”与“Upcoming Shows”整合在右侧，形成独立的“发现”区域。
- 增加了“最近播放”模块，提升日常使用效率。
- 整体色彩从单调深色转变为初音主题的蓝绿+粉紫高亮，更具辨识度。
