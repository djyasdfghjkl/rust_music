# 播放器界面设计方案 · 沉浸式·未来感·初音主题

## 一、核心设计理念

**打破“顶部专辑图+中部控件+底部列表”的庸常模式**，采用**全动态背景 + 模煳分层 + 手势优先**的设计语言，让播放器界面本身就是一场视觉演出。

- **沉浸感最大化**：当前播放歌曲的专辑封面（或主题插画）作为全窗口背景，通过深色蒙版+动态模糊保证可读性。
- **信息层级极简**：只展示最关键的内容——歌曲名、艺术家、专辑、进度条、歌词（可选），次要控件默认隐藏或收纳为悬浮按钮。
- **玻璃质感进化**：不只是简单的毛玻璃，而是**多层次的流光溢彩**（跟随音乐频谱或节奏产生细微的色彩偏移）。
- **初音未来元素融合**：保留初音绿（#39C5BB）和粉红（#FF9EC5）作为高亮色，但在背景中融入葱色光斑、双马尾轮廓的粒子效果。

## 二、整体布局架构（无边框全屏窗口）

| 区域 | 位置 | 内容 | 默认可见性 |
|------|------|------|------------|
| **主视觉背景** | 全窗口底层 | 当前歌曲的专辑封面/初音主题插画，叠加动态高斯模糊（blur 20px） + 暗色渐变蒙版 | 始终可见 |
| **中央控制区** | 窗口中心（偏下） | 播放控制按钮（大号）、进度条、时间显示 | 始终可见 |
| **歌曲信息区** | 中央上方 | 歌曲名（大字号）、艺术家、专辑名（次要文字） | 始终可见 |
| **歌词展示区** | 窗口左侧/右侧（可切换） | 当前歌词行高亮，其他行半透明，支持滚动 | 默认收起，鼠标滑入或点击歌词图标展开 |
| **悬浮工具栏** | 窗口四角（悬浮） | 返回首页、音量调节、播放列表、更多操作（循环模式、收藏等） | 鼠标移入窗口边缘时浮现，平时半透明 |
| **底部迷你进度条** | 窗口底部边缘 | 细条进度线，可快速跳转 | 始终可见（极简） |

## 三、新颖交互与视觉亮点

### 3.1 全动态背景（Living Background）
- 专辑封面不再是一张小图，而是**铺满整个窗口**，经过强模糊和暗化处理后作为氛围底图。
- **可选增强**：从封面图像中提取主色调，动态改变控件高亮色和进度条渐变色（例如天空蓝 → 初音绿 → 紫罗兰，跟随歌曲情绪）。
- 初音主题下，背景中随机飘浮**绿色的葱形光斑**或**「01」数字矩阵**，缓慢移动，增加科技感。

### 3.2 极简控制栏（Ultra-thin Controls）
- 播放/暂停按钮：一个**大号毛玻璃圆形**，中心为Miku的Q版头像或音符图标，点击时有微妙的弹性动画。
- 上一首/下一首：两个小型圆角矩形，位于大按钮两侧，鼠标悬浮时才完全显现（平时半透明）。
- 进度条：不再是传统横条，而是一条**发光的细线**，游标为一个**发光的葱**（绿色小光点），拖拽时显示时间气泡。

### 3.3 歌词展示的创新方式
- 默认歌词隐藏，用户**将鼠标移动到窗口右侧边缘**，右侧会滑出一块半透明毛玻璃面板，显示当前歌曲的滚动歌词。
- 歌词行高亮使用**渐变色**（初音绿 → 粉红），并且每个字符可跟随音乐节奏轻微跳动（仅视觉特效，非3D）。
- 支持手动拖动歌词区域调整大小，或固定到左侧。

### 3.4 手势与键盘优先
- 因为是无边框窗口，支持**鼠标拖拽窗口任意位置移动**（除控件外）。
- **键盘快捷键**：空格（播放/暂停）、←/→（跳转5秒）、↑/↓（音量）、L（显示/隐藏歌词）、F（收藏）。
- **鼠标滚轮**在窗口左侧调节音量，在右侧调节亮度（如果实现）。

### 3.5 音乐可视化（可选，但提升新颖度）
- 在进度条上方或背景中的某个区域，绘制**实时频谱**，频谱条采用初音绿渐变，高度随音乐变化。
- 频谱可呈现为圆形、波浪形或粒子环形，更符合“未来感”。

## 四、具体视觉规范

### 4.1 配色（基于初音主题，但更前卫）

| 元素 | 色值 | 说明 |
|------|------|------|
| 背景蒙版 | `linear-gradient(135deg, rgba(0,0,0,0.7), rgba(30,20,40,0.85))` | 覆盖在模糊封面之上 |
| 主要文字 | `#FFFFFF` | 100%白，加轻微发光 |
| 次要文字 | `rgba(255,255,255,0.65)` | 艺术家、专辑 |
| 高亮色 | `#39C5BB`（初音绿） | 进度条填充、播放按钮悬停 |
| 强调色 | `#FF9EC5`（粉红） | 收藏图标、当前歌词行 |
| 毛玻璃背景 | `rgba(20,20,40,0.4)` + `backdrop-filter: blur(12px)` | 控制栏、歌词面板 |

### 4.2 字体与尺寸

- 歌曲名：36px，字重700，使用`Poppins`或`Inter`，允许换行。
- 艺术家/专辑：16px，字重400，间距宽松。
- 时间码：14px，等宽字体（`JetBrains Mono`）。
- 歌词：18px，行高1.6，当前行放大至22px并加粗。

### 4.3 圆角与边缘

- 所有控件均采用大圆角（`24px`或圆形），消除生硬边角。
- 窗口本身支持圆角（`16px`），与Tauri配置匹配。

# 播放器界面完整设计方案 · Miku Wave Player

## 一、设计哲学与创新突破

### 1.1 核心理念
- **内容即背景**：当前播放的歌曲封面/主题插画不再是角落的装饰，而是铺满整个窗口的氛围底色。
- **极简至无形**：常用控件只在需要时浮现，平时几乎隐形，让用户专注于音乐与歌词。
- **触感与动效**：所有交互都伴随细腻的物理动画（弹性、惯性、光效），让界面“活”起来。
- **初音未来灵魂**：将初音绿、粉红、葱元素、01标识等融入每一个细节，但不喧宾夺主。

### 1.2 与传统播放器界面的差异

| 传统设计 | 本方案设计 |
|----------|------------|
| 固定专辑封面小图 | 全屏动态模糊背景 + 色彩提取 |
| 控件密集排列在底部 | 悬浮工具栏，平时隐藏，边缘触发 |
| 歌词占据固定区域 | 侧滑面板，按需呼出，优雅缩放 |
| 静态无变化 | 背景光效随音乐频谱脉动 |
| 传统进度条 | 发光细线 + 创意游标（葱形） |
| 无全局手势 | 拖拽移动窗口、滚轮调音、边缘拖拽 |
| 窗口固定大小 | 全窗口无边框，任意比例适配 |

## 二、整体布局架构（全窗口无边框）

窗口背景层级（从底到顶）：

```
┌─────────────────────────────────────────────────────────┐
│   Layer 0: 全窗专辑封面（模糊+暗化）                      │
│   Layer 1: 动态光效（颜色提取+频谱粒子）                  │
│   Layer 2: 主内容区域（歌曲信息 + 控制核心）              │
│   Layer 3: 悬浮工具栏（平时半透明，hover显现）            │
│   Layer 4: 歌词面板（滑动出现，可拖拽位置）               │
│   Layer 5: 底部迷你进度条（始终可见，细线）               │
└─────────────────────────────────────────────────────────┘
```

### 2.1 窗口基础属性
- 无边框、可拖拽、圆角16px、支持深色/透明效果。
- 默认尺寸：1000×700，支持自由缩放，内容自适应居中。
- 背景默认黑色，加载封面后替换为封面模糊图。

## 三、模块详细设计

### 3.1 动态背景系统

**封面处理流程**：
1. 当前歌曲加载时，获取其专辑封面（若没有则使用默认初音主题图）。
2. 将封面图片铺满整个窗口，应用高强度高斯模糊（`blur(24px)`）。
3. 叠加一层半透明渐变蒙版：`linear-gradient(135deg, rgba(0,0,0,0.6), rgba(30,20,40,0.8))`。
4. 从封面中提取主色调（如天空蓝），将工具栏、进度条的高亮色动态调整为该色调的互补色或协调色（默认仍保留初音绿为主，但可微调）。

**动态光效**：
- 在背景上随机生成缓慢飘浮的**葱形光斑**（半透明绿色圆形光晕）和**「01」数字矩阵**（极淡的重复纹理）。
- 可选：接入音乐频谱分析，让光斑的亮度或大小随低音节奏脉冲。

### 3.2 歌曲信息展示区（中央上部）

- **位置**：窗口垂直方向40%处，水平居中。
- **歌曲名**：最大字号44px，字重700，白色，带极淡文字阴影；若过长则自动缩放字体或换行（最多两行）。
- **艺术家名**：位于歌曲名下方8px，字号18px，字重500，颜色`rgba(255,255,255,0.75)`。
- **专辑名**：位于艺术家名下方4px，字号14px，字重400，颜色`rgba(255,255,255,0.5)`，前面可加“专辑·”前缀。
- **动画**：切换歌曲时，歌曲信息区域有淡入+向上位移20px的过渡效果（0.3s缓动）。

### 3.3 控制核心区

位置：窗口垂直方向70%处，水平居中。

**播放/暂停按钮**：
- 直径72px的圆形毛玻璃按钮（`background: rgba(255,255,255,0.15); backdrop-filter: blur(8px);`）。
- 中心图标：播放时显示暂停符号（||），暂停时显示播放三角符号（▶）。
- 图标颜色：初音绿（`#39C5BB`），hover时变为粉红（`#FF9EC5`）。
- 点击动画：轻微缩放（0.95→1.0），弹性曲线。

**上一首/下一首**：
- 位于播放按钮左右两侧，距离48px。
- 按钮直径48px，同样毛玻璃圆角，图标大小20px。
- 平时透明度0.6，hover时变为1.0。
- 点击时伴随向左右微弹的动画。

**进度条**：
- 位于播放按钮下方24px处，宽度400px（可随窗口缩放比例变化）。
- 外观：一条细线（高度2px），底色`rgba(255,255,255,0.2)`，已播放部分为渐变色（初音绿 → 粉红）。
- 游标：一个发光的葱形小圆点（直径12px，绿色光晕），鼠标移入进度条时显示，拖拽时放大至16px并显示时间气泡。
- 拖拽交互：鼠标按下后全局移动，实时更新播放位置，释放后跳转。
- 实时时间显示：在进度条左右两侧显示当前播放时间（`mm:ss`）和总时长，字号12px，等宽字体，颜色`rgba(255,255,255,0.6)`。

### 3.4 悬浮工具栏（边缘触发）

- **默认状态**：四角各有半透明小圆点或图标，透明度0.3。
- **触发方式**：鼠标移入窗口边缘（上下左右各20px区域），相应角落的工具栏展开为完整按钮组。
- **左上角**：返回首页图标（🏠）+ 应用菜单（···）。
- **右上角**：音量控制（扬声器图标+滑条）、播放模式（循环/随机/单曲循环）、收藏心形。
- **左下角**：播放列表队列（☰ 图标）。
- **右下角**：歌词显示开关（📄 图标）、全屏切换、窗口关闭/最小化（系统按钮）。
- **交互**：工具栏展开时有淡入+轻微位移动画，移出边缘区域后延迟2秒自动收起。

**音量控制细节**：
- 滑条垂直或水平？采用水平小滑条，长度80px，高度3px，底色白半透明，填充色初音绿。
- 鼠标悬停在音量图标上即显示滑条，调整时实时改变系统音量。

**收藏按钮**：
- 心形图标，未收藏时为空心白边，收藏后变为粉红色实心并有轻微跳动动画。

### 3.5 歌词面板（创新滑动设计）

这是本方案的**核心亮点**，彻底告别传统歌词块。

**默认状态**：完全隐藏，不占用任何空间。

**呼出方式**：
- 鼠标移动到窗口**右侧边缘**（或点击右下角歌词图标），右侧会滑出一块半透明毛玻璃面板。
- 面板宽度：280px，高度：窗口高度的80%，垂直居中。
- 动画：从右向左滑入，伴随0.2s缓动。

**面板内容**：
- 头部显示“歌词”标题，可点击固定面板（使其不自动收回）。
- 歌词文本垂直滚动，当前播放行居中并高亮。
- 高亮样式：字体颜色初音绿，字号20px，加粗，左侧带粉红色竖线；其他行颜色`rgba(255,255,255,0.5)`，字号16px。
- 支持手动滚动歌词，点击任意行可跳转到该行对应时间（若LRC有时间轴）。

**歌词数据来源**：
- 优先读取同目录下的 `.lrc` 文件（与歌曲文件名相同）。
- 若无LRC，则显示静态歌词文本（按段落分割），用户可手动滚动阅读。
- 对于参考图片中的歌词片段，可预先嵌入或通过解析显示。

**关闭方式**：
- 鼠标移出右侧边缘区域（且未固定在面板上）后3秒自动收回。
- 点击面板外的任意位置收回。
- 点击面板上的“固定”图标可保持一直显示。

**增强体验**：
- 歌词面板背景为半透明毛玻璃（`rgba(20,20,40,0.6)` + `backdrop-filter: blur(16px)`），边框绿色微光。
- 面板边缘可拖拽调整宽度（200~400px）。

### 3.6 音乐可视化（频谱）

- **位置**：可放置在进度条上方（细条状），或背景中作为氛围元素。
- **样式**：一组垂直的细条（或圆形粒子），高度随音乐频率变化，使用初音绿到粉红的渐变。
- **显示条件**：默认显示小频谱在进度条正上方（高度30px，宽度与进度条一致），用户可在悬浮工具栏中开关。
- **技术提示**：前端通过Web Audio API获取实时频域数据，绘制Canvas动画。

### 3.7 底部迷你进度条

- **位置**：窗口最底部边缘，一根细线（高度1px），宽度100%。
- **作用**：始终显示当前播放进度占整首歌的比例，点击任意位置可快速跳转。
- **视觉**：底色深灰，填充色初音绿，游标不显示（因为太细），鼠标移入时线变粗至3px并显示时间标记。
- **优势**：不占用主要空间，但提供快速跳转的便利。

## 四、交互与动效细节

### 4.1 切换歌曲动画
- 背景封面渐变过渡（交叉淡变，0.4s）。
- 歌曲信息区域向上淡出，新信息从下方淡入。
- 进度条归零，时间重置。
- 如果歌词面板打开，歌词自动滚动到新歌曲的第一行（或显示静态歌词）。

### 4.2 播放/暂停动画
- 按钮中心的图标切换时有旋转或缩放效果。
- 频谱动画播放时开始运动，暂停时冻结。
- 可选：背景光效的脉冲随播放状态变化（播放时脉动更明显）。

### 4.3 音量调节反馈
- 调节时屏幕右上角出现临时音量百分比气泡，1秒后消失。
- 音量图标随音量大小改变填充样式（低音量时半格，静音时带斜线）。

### 4.4 窗口拖拽与无边框细节
- 整个窗口除控件外均可拖拽移动（通过 `-webkit-app-region: drag`）。
- 按钮区域需设置 `-webkit-app-region: no-drag`。
- 圆角窗口边缘需配合Tauri配置实现抗锯齿。

## 五、初音未来主题元素融合

## 十、技术实现提示（仅设计参考，不涉及代码）

- **前端**：Vue/React + TypeScript，Web Audio API（频谱），Canvas（动画）。
- **后端**：Tauri + Rust，提供音频文件流和元数据解析。
- **歌词解析**：前端解析 `.lrc` 文件，或通过IPC读取本地歌词文件。
- **封面处理**：Rust后端提取内嵌封面图片，缓存为临时文件，前端通过 `blob:` URL 加载并应用 CSS 滤镜。
- **无边框圆角**：Tauri 配置中设置 `"transparent": true` 和 `"decorations": false`，前端通过 CSS 设置 `border-radius` 和 `box-shadow`。
